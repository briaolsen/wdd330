Chapter 8 - Transforms and Transitions

Transforms
  • Translation - move elements left, right, up or down
  • "transform: translate(x,y);" "transform: translateX( px);" "transform: translateY( px);"
  • Scaling - scales an element by defined factors horizontally and vertically, one value will scale the whole thing
  • "transform: scale(x,y);" "transform: scale();" "transform: scaleX();" "transform: scaleY();"
  • Rotation - rotates an element around the point of origin, positive = clockwise, negative = counterclockwise
  • "transform: rotate( deg);"
  • Origin - can control the origin from where the transforms are applied
  • "transform-origin: 50% 50%"
Transitions
  • Transitions - allow the values of CSS properties to change over time = simple animations
  • transition-property: defines the CSS properties of the element that should be transitioned, "all" being default
  • transition-duration: sets how long the transition will take
  • transition-timing-function: controls the pace of the transition in more detail (ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier)
  • transition-delay: delay before the transition begins
  • "transition: property, duration, timing-function, delay"
Animations
  • Keyframes - used to animate an element, use @keyframes rule, include @-webkit-keyframes
  • "@-webkit-keyframes myAnimation { animation here }" "@keyframes myAnimation {animation keyframes}"
  • animation-name: used to attach an animation to an element - appearDisappear
  • animation-duration: length of time it takes for one iteration
  • animation-timing-function: how the animation will progress over its duration (ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier, start-stop, step-end, steps(number, direction))
  • animation-iteration-count: lets you define how many times the animation will play through (1, infinite)
  • animation-direction: normal is from 0 to 100% (reverse, alternate, alternate-reverse, normal)
  • animation-delay: define how many milliseconds or seconds to wait before the browser begins the animation
  • animation-fill-mode: defines what happens before the first animation iteration begins and after the last animation iteration concludes (none, forwards, backwards, both)
  • animation-play-state: defines whether the animation is running or paused
  • animation: shorthand version
