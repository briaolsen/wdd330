Form Properties and Methods
  • form.submit() - submit the form automatically, won't trigger the submit event
  • Can submit with: button type='submit', input type='submit' value='Submit', input type='image' src='button.png'
  • form.reset() - reset all the form controls to initial values
  • Can reset with: button type='reset'
  • form.action - set the action attribute of a form to be sent to a different URL: form.action = '/an/other.url'

Chapter 12 - Object-Oriented Programming in JavaScript

  • Encapsulation - the programming logic implemented inside an object and user does not know how it's done
  • Polymorphism - various objects can share the same method but implement them differently or more specifically
  • Inheritance - an object and its properties and methods can be inherited and added to
  • Classes - classes are used to definite a blueprint for an object
Classes
  • Constructor - constructor(params) {} example: class Dice { constructor(sides=6){this.sides=sides;}} const blueDice = new Dice(20);
  • Static Methods - keyword used in declarations, aka "class methods", called by the class and not by the instances of a class
  • Prototype - default property that is shared by every instance of the class added to the class declaration, can be accessed using .prototype, .constructor.prototype, Object.getPrototypeOf(var), ._proto_, can be reassigned using =, can add prototypes (or functions) later Class.prototype.variable =
  • Public Methods - default public, can be seen and changed
  • Private Methods - can keep variables private so they cannot be accessed or changed, use getter & setter methods to return or change in a controlled way
  • Prototype Chain - chain of inheritance
  • Inheritance - can inherit using extends in class declaration, use super() to refer to parent class
  • Polymorphism - different objects can have the same method but implement it in different ways, objects can override a method with more specific implementation
  • Adding Methods to Built-In Objects - you can add more methods to the prototype of built-in objects (Number, String, Array) - "monkey-patching"
Property Attributes and Descriptors
  • value - the value of the property, undefined by default
  • writable - boolean value shows if a property can be changed or not
  • enumerable - boolean value shows if a property will shwo up when an object is displayed in a for in loop
  • configurable - boolean value shows whether you can delete a property or change any of its attributes
  • getters and setters - use to change and return properties
Creating Objects from Other Objects

Can create new objects based on another object that acts as a blueprint or prototype instead. Can avoid using classes.

Object-Based Inheritance

Can use objects like a 'super-class' and become a prototype for another object. This will have all the properties and methods but with some extra methods. Mixins - adding properties and methods of some objects to another object without using inheritance - use Object.assign() method - assign to first argument all the properties of the other arguments.

Mixins
  • A mixin is a way of adding properties and methods of some objects to another object without using inheritance.
  • can add in a lot of properties at once - mixin(wonderWoman,{ name: 'Wonder Woman', realName: 'Diana Prince' });
  • Create a copy() method - deep copy
  • Factory Function - function that can be used to return an object, uses copy() function to make a copy then uses the mixin() function to augment any properties and methods of any bojects athat are provided as arguments
  • Mixin() allows us to encapsulate properties and methods in an object then add them to other objects without the overhead of an inheritance being created
  • Methods can be chained together and call all three at once ...
  • Make that = this to keep this from getting lost
  • bind(this) - if this is provided as an argument to bind while its still in scope, this will stay this
  • Use Arrow Functions this stays this
  • Possible to borrow methods from objects without having to inherit all their properties and methods
  • Borrow Array methods - you can borrow methods from arrays, can use slice in arguments
