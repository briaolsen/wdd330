Javascript: Novice to Ninja

Chapter 10 - Testing and Debugging

Errors, Exceptions, and Warnings
  • Errors - caused when something goes wrong in a program; system, programmer, or user
  • Exception - error that produces a return value that can be used by the program to deal with the error
  • Stack Traces - produced by an exception; a sequence of functions or method calls that lead to the error
  • Warning - there's an error but not enough to crash the program
  • Strict Mode - produces more exceptions and warnings and prohibits deprecated features; use to spot errors early and forces better code; 'use strict'; in JS file at top or inidividual functions
  • Linting Tools - can be used to test the quality of JS code beyond strict mode (JS Lint, JS Hint, ES Lint), useful for enforcing a style guide
  • Feature Detection - use an if statement to check if something exists before calling a method
Debugging in the Browser
  • alert() - show a dialog at certain points of the code, stops it until ok is clicked
  • console.log() - use to log variable values, does not stop the program
  • console.trace() - will log an interactive stack trace to the console
  • debugger - creates a breakpoint that will pause the program, can hover over variables to see what they are at the time
  • error object - can be created my the program or when an exception occurs; RangeError, ReferenceError, SyntaxError, TypeError, URIError, InternalError - can be used as constructors; have properties: name (returns type of error), message (returns a description of the error), stack (returns a stack trace)
  • throw - used to throw your own exceptions, should throw an error object, can give it your own message
  • try/catch/finally - wrap code in try/block to catch an exception, catch block will handle the problem; finally block will always be executed after try & catch
Test-Driven Development

Red-Green-Refactor cycle

  1. Write tests (initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features
Testing Frameworks
  • Jest - TDD framework by Facebook, provides helper methods for common test assertions; npm install -g jest; example in script tags
